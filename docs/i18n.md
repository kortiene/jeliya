# Internationalization

The language plan of record, and the engineering rules that keep the app
translatable. Terminology and French scoping live in `glossary-fr.md`; this
file records the decisions and the mechanics.

## Decisions (settled 2026-07-07/08)

- **French ships at desktop launch**, full-catalog in one release — the
  three-column desktop layout makes a partially translated window
  unacceptable (see `glossary-fr.md`, decision 5).
- **Bambara (`bm`) is a fast-follow** via community translation on
  self-hosted Weblate. CLDR gives `bm` a single plural category (no plural
  forms needed) but below-basic coverage — under a `bm` text locale the
  formatting locale stays `fr`/`en` (decision 4 makes this expressible).
- **N'Ko (`nqo`) is pending an on-device rendering test** (RTL script,
  cursive joining; macOS bundles Noto Sans NKo, Windows/Linux need it
  bundled). `nqo` is absent from `package:intl`, and **`bm_Nkoo` is not in
  ICU — never target it as a locale**.
- **Daemon/CLI output stays English** (glossary decision 1). The UI maps
  `{code, message, hint}` to translated copy client-side; raw daemon text is
  only ever the collapsed "Technical details" disclosure, the Settings
  diagnostics card, and the diagnostics report (support artifacts).
- **Text locale ≠ formatting locale** from day one (glossary decision 4).

## Engineering rules (enforced by `scripts/i18n-gate.mjs`)

1. **All user-visible copy lives in `app/lib/src/l10n/arb/app_en.arb`** —
   stable `<area><Key>` names, an `@description` written for translators on
   every entry (enforced by `required-resource-attributes`). `flutter
   gen-l10n` generates `AppStrings` into `lib/src/l10n/gen/` (committed,
   never hand-edited); widgets resolve it AT RENDER TIME via
   `context.strings` — never cached in state, never via `lookupAppStrings`
   in lib/ (the gate rejects it). Non-copy strings (glyphs, URLs, shell
   commands, wire-format examples) live in `l10n/tokens.dart` and never
   reach translators. Deliberate exceptions carry an
   `// i18n-exempt: <reason>` marker.
2. **No sentence assembly in widget trees.** A sentence with styled or
   interactive segments is ONE catalog message whose `{slot}` placeholders
   the call site passes back as literal `'{marker}'` strings, rendered via
   `app/lib/src/widgets/template_text.dart` — translators reorder slots like
   any placeholder. Lists of independent facts may be joined with the
   decorative `Tokens.metaSep`; sentences may not.
3. **No wire values as display text.** Protocol enums (roles, member
   statuses, peer paths, daemon modes, connection states) render through
   the `WireDisplay` extension (`l10n/wire_display.dart`); error copy
   through `ErrorDisplay.friendlyError` (`l10n/error_display.dart`) —
   resolved per build. Unknown values pass through raw (forward compat).
   Display labels and wire values are never the same constant.
4. **All display formatting goes through `JeliyaFormats`**
   (`app/lib/src/format.dart`, via `context.formats`): bytes, clock, day
   labels, percent, and relative time. Numeric/calendar conventions use the
   EXPLICIT formatting locale; vocabulary such as byte units, Today, and
   “ago” follows the text catalog. `prettyLabel`/`extOf` stay pure. Never
   format numbers/dates inline.
5. **`jeliya_protocol` composes no user-visible English.** Conventions emit
   structured facts (e.g. `JoinProgress.phase/attempt/retryDelay`) and the
   app narrates them. Same rule app-side: `DaemonSession` stores structured
   `BootStage` facts, never composed copy. Exception messages are technical
   detail, not UI copy.
6. **Tests assert copy via the shared `en` catalog instance**
   (`test/helpers.dart`), not literals, so translation work never breaks
   them. `test/l10n_parity_test.dart` is GENERATED by
   `scripts/gen-l10n-parity.mjs` — re-run it after every ARB edit. (Fixture
   data — room names, message bodies — stays literal; it is data, not copy.)
7. **All-caps treatments use `String.toUpperCase()` at render time** on an
   already-translated string. That is Unicode-correct and safe for the
   target locales (French/Bambara keep their accents); if a locale with
   locale-dependent casing (Turkish, Greek) is ever added, casing must move
   into the catalog since Dart has no locale-aware case mapping.

## React implementation and fallback

The web client mirrors the same contract without a runtime i18n dependency:
`ui/src/l10n/catalog.ts` defines the complete typed shape, `en.ts` and `fr.ts`
implement it, and components resolve it at render time through `useStrings()`.
Dates, numbers, counts, percentages, relative time, and byte sizes go through
`useFormats()`. Structured sentences with styled values use `Template`; wire
values and friendly errors use the dedicated display maps. Tickets, identity
ids, paths, signed content, commands, and glyphs remain data/tokens and are
never passed through translation.

Settings persists two independent browser preferences:

- `jeliya.textLocale`: unset means follow the first supported primary language
  in `navigator.languages`, falling back to English.
- `jeliya.formattingLocale`: unset means follow the first platform locale that
  `Intl` supports, even when that locale has no text catalog; if none is usable,
  it falls back to the resolved text locale.

Both preferences apply live. A browser `languagechange` event re-resolves an
unset preference without a restart. `scripts/check-ui-i18n.mjs` blocks missing,
empty, untranslated, or typographically invalid catalog entries and any new
user-visible component literal outside the catalog; `npm run i18n:report` is a
non-failing inventory view of that same literal scan.
